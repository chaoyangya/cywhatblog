给公司写的一个工具平台，先部署到自己的服务器上进行测试，真是困难重重，此文章仅记录自己踩坑过程和搭建教程，不保证你能按照教程成功搭建，但能把坑提前暴露出来，避免大家重复踩坑！

我的环境

  1. 部署服务器 ：CentOS Linux release 7.8.2003 (Core)
  2. Python：3.7.0
  3. Django ：1.11.7
  4. Nginx：1.16.1
  5. Uwsgi：2.0.19.1
  6. 操作机：mac + iterm2

1、上传/clone项目至服务器

1

$ git/xftp均可

2、安装python3.7.0

安装python3.7.0：

1

$ wget http://www.python.org/ftp/python/3.7.0/Python-3.7.0.tgz

解压

1

$ tar -xvzf Python-3.7.0.tgz

进入文件夹

1

$ cd Python-3.7.0

执行配置文件

1

$ ./configure --with-ssl

编译

1

$ make

安装

1

$ sudo make install

建立软连接

1
2

$ ln -s /usr/local/python3/bin/python3.7 /usr/bin/python3
$ ln -s /usr/local/python3/bin/pip3.7 /usr/bin/pip3

检测python3版本

1

$  python3 -V

安装虚拟环境

1、安装虚拟环境

1

$ pip3 install virtualenv

2、新建虚拟环境文件夹

1

$ mkdir pyenv

3、进入虚拟环境文件夹

1

$ cd pyenv

4、安装虚拟环境

1

$ virtualenv  -p python3 myenv

5、激活虚拟环境

1

$ source /myenv/bin/activate

6、安装项目所需组件

1

$ pip3 install -r requriements.txt

7、pip安装依赖太慢使用douban源

1

$ pip install xx -i http://pypi.douban.com/simple/

安装nginx

1、安装依赖

1

$ yum -y install gcc pcre-devel zlib-devel openssl openssl-devel

2、下载nginx

1

$  wget http://nginx.org/download/nginx-1.16.1.tar.gz

3、新建文件夹

1

$ mkdir /usr/loacl/nginx

4、解压nginx包

1

$ tar -zxvf nginx-1.16.1.tar.gz

5、进入解压后的nginx目录

1

$ cd nginx-1.16.1/

6、编译

1

$ ./configure  --prefix=/usr/local/nginx

7、安装

1

$ make && make install

8、修改nginx.conf配置即可

1

$ vim /usr/local/nginx/conf/nginx.conf

疑难解答

1、启动uwsgi.ini报错

WARNING: Can’t find section “uwsgi” in INI configuration file uwsgi.ini

1

$ 原因：uwsgi.ini头部没有添加[uwsgi] 头部文件 添加完成以后即可

2、部署项目完成以后，而且静态资源也已经迁移，访问资源还是403，修改nginx的使用用户为root即可

1

$ vim /usr/local/nginx/conf/nginx.conf

1

#有些nginx安装完成后这句是注释的 打开即可

3、nginx部署完成后，出现-bash: nginx: command not found 添加nginx的环境变量即可

1

$ vim /etc/profile

在末尾加入：

1
2

$ PATH=$PATH:/var/local/nginx/sbin
$ export PATH

然后让配置生效

1

$ source /etc/prifile

然后就可以使用nginx快捷命令了
